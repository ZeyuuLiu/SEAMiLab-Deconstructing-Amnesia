## 2. Required Inputs

Generation probe input:

1. `Q`
2. `C_oracle` (from LOCOMO sample build for current query)
3. `A_oracle` (generated by original memory-system model under `C_oracle`)
4. `A_gold`
5. `task_type`

In code:

1. `GenerationProbeInput.question`
2. `GenerationProbeInput.oracle_context`
3. `GenerationProbeInput.answer_oracle`
4. `GenerationProbeInput.answer_gold`
5. `GenerationProbeInput.task_type`

## 3. Adapter Integration

Adapter protocol:

1. `GenerationAdapterProtocol.generate_oracle_answer(run_ctx, query, oracle_context)`

Entrypoint:

1. `evaluate_generation_probe_with_adapter(sample, adapter, run_ctx, cfg)`

This guarantees `A_oracle` comes from the original memory-system response model under oracle context.

## 4. Decision Logic

### Step 1

Use adapter to generate `A_oracle` with `C_oracle`.

### Step 2

Compare `A_oracle` with `A_gold`:

1. Rule mode baseline (deterministic fallback)
2. Optional LLM-assisted judge for smarter classification

State:

1. `PASS`
2. `FAIL`

FAIL subclasses:

1. `GH` (NEG no-abstain)
2. `GF` (POS ungrounded incorrect)
3. `GRF` (POS grounded incorrect)

## 5. LLM-Assisted Judgement

Optional API:

1. `llm_judge_generation_answer(...)`

Returns strict JSON:

1. `correct`
2. `substate` (`GH|GF|GRF|NONE`)
3. `grounded`
4. `reason`

If LLM call fails, rule fallback remains active.

## 6. Evidence Output

Generation probe evidence includes:

1. `answer_oracle`
2. `answer_gold`
3. `llm_judgement` (if used)
4. grounding overlap metadata
5. textual reason

## 7. Independent Test Script

Script:

1. `scripts/test_generation_probe.py`

Cases:

1. POS pass
2. NEG pass
3. NEG GH
4. POS GF
5. POS GRF
